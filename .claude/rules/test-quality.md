# テスト品質ルール

## 禁止事項

### テストの改ざん禁止

- [ ] 既存テストの assertion を弱めない
- [ ] テストケースを削除しない（非推奨化は別途相談）
- [ ] `skip` / `only` を本番コミットに含めない
- [ ] モックで実際の動作を隠蔽しない

### 形骸化テスト禁止

```typescript
// NG: 何もテストしていない
it("should work", () => {
  expect(true).toBe(true);
});

// NG: 実装をそのままコピー
it("should calculate", () => {
  const result = calculate(2, 3);
  expect(result).toBe(calculate(2, 3)); // 意味がない
});
```

## 必須事項

### テスト追加時

- 正常系・異常系の両方をカバー
- エッジケースを考慮
- テスト名は期待動作を明確に記述

### テスト修正時

- 修正理由をコミットメッセージに記載
- カバレッジが下がらないことを確認
